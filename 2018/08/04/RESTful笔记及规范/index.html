RESTful笔记及规范

发表于 2018-08-04

前言：REST是一种风格参考，不是硬性要求。

Roy Fielding的毕业论文。这哥们参与设计HTTP协议，也是Apache Web的co-founde；所以提出的设计思想是有保障的。

论文地址：https://www.ics.uci.edu/~fielding/pubs/dissertation/rest_arch_style.htm

我认为目前RESTful风格所解决的问题是:

clients:

  • IOS
  • Android
  • web
  • Other

建立在RESTful还未形成的时候或许不同的设备需要提供不同的API，所以RESTful可以标准化整个API，并且需要为多个client提供结果，那变相的也必须要保证API的健壮性和可靠性

RESTful是一种风格；通过与传统RPC或Struts的实际效果来总结RESTful的特点：

  • 基础操作符 GET POST DEL PUT
  • HTTP中的Headers的利用率
  • URI的命名
  • 解除JSP方式，前后分离
  • 分离后的交互数据结构
  • URI是名词不是动词，推荐复数
  • 安全的URI

以上几点就能看出，满足这几个特点基本可以对HTTP的请求形成一套标准化
